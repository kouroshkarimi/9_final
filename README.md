# Collaboration Demo Project

This project is used to practice Git and GitHub collaboration workflows.

## How to contribute
- Open an issue describing the problem or feature
- Wait for feedback from maintainers
- Create a branch with a descriptive name
- Make your changes following the coding standards
- Submit a pull request and link it to the issue
- Respond to review comments and make necessary changes
- Once approved, your PR will be merged

## Coding Standards
- Keep changes focused on one issue per PR
- Write clear commit messages
- Add comments for complex logic
- Ensure CI passes before requesting review

## Testing Requirements
- All new features should include tests
- Run tests locally before submitting PR
- CI must pass for PR to be merged
